🛡️ Data Quality Monitoring

Embedded DQ in DLT Pipeline

Data quality expectations are CODE in the DLT pipeline, not external validation scripts. Records failing expectations are QUARANTINED (not dropped silently), giving actuaries complete visibility into data issues.

6 Data Quality Expectations

# Expectation Rule Action
1 valid_dates Expiry date > Inception date DROP
2 valid_state State in (NSW, VIC, QLD, WA, SA, TAS, ACT, NT) DROP
3 positive_premium Annual premium > 0 DROP
4 incurred_gte_paid Incurred cumulative ≥ Paid cumulative DROP
5 valid_lodgement Lodgement date ≥ Accident date DROP
6 valid_status Status in (open, closed, reopened) DROP

❌ Traditional ETL (SAS/Excel)

  • ✗ Bad data dropped silently
  • ✗ No visibility into failures
  • ✗ Manual validation in spreadsheets
  • ✗ DQ as afterthought
  • ✗ No audit trail

✅ Databricks DLT

  • ✓ Records QUARANTINED (not dropped)
  • ✓ Complete visibility in DQ table
  • ✓ DQ expectations as CODE
  • ✓ Embedded in pipeline
  • ✓ Full audit trail in event log
# Example DLT DQ Expectation (Python)

@dlt.table(name="claims_clean")
@dlt.expect_or_drop("incurred_gte_paid",
                    "incurred_cumulative >= paid_cumulative")
@dlt.expect_or_drop("valid_lodgement",
                    "lodgement_date >= accident_date")
def claims_clean():
    return spark.sql("""
        SELECT c.*, p.product, p.state
        FROM actuary_corpfin.bronze.claims_transactions_raw c
        LEFT JOIN actuary_corpfin.bronze.policy_raw p
            ON c.policy_id = p.policy_id
    """)

⏰ Delta Lake Time Travel

Regulatory Audit: "What did we know when?"

Delta Lake automatically versions every change to your tables. Query data as of any prior timestamp or version number with a single SQL query. No need to re-run jobs or maintain manual snapshots!

Time Travel Use Cases

📋 Regulatory Compliance

"What triangle did we submit to APRA last quarter?"

SELECT *
FROM development_triangles
TIMESTAMP AS OF '2025-09-30'

📊 Variance Analysis

Compare current vs prior quarter reserve estimates

SELECT
  current.accident_year,
  current.incurred as current_est,
  prior.incurred as prior_est
FROM development_triangles current
JOIN development_triangles
  VERSION AS OF 5 prior
  ON current.accident_year =
     prior.accident_year

🔄 Mistake Recovery

Restore accidentally deleted or modified data

CREATE OR REPLACE TABLE
  development_triangles
AS SELECT *
FROM development_triangles
VERSION AS OF 10

🔬 Reproducibility

Recreate any prior analysis exactly as it was

-- Recreate Q3 2025 analysis
SELECT *
FROM development_triangles
TIMESTAMP AS OF
  '2025-09-30T23:59:59Z'
